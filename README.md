# 📘 Daily Programs Repository

Welcome to the **Daily Programs** repository! 🚀  

This repository is dedicated to solving **daily coding challenges**, **random problems**, and **practice questions** from platforms like:
- GeeksforGeeks  
- LeetCode  
- HackerRank  
- CodeChef  
- And more...

The goal is to **build consistency**, **improve problem-solving skills**, and **share knowledge collaboratively**.

---

## 📂 Repository Structure

Each problem must be organized in its **own directory**:

```
dailyprograms/
│
├── problem-name-1/
│   ├── solution.cpp
│   ├── explanation.md (optional)
│
├── problem-name-2/
│   ├── solution.py
│
└── ...
```


- Each folder = one problem  
- Include:
  - Solution code  
  - Optional explanation or approach

## 🌱 Contribution Guidelines

This is a **collaborative repository**, but with structured rules to keep it clean and scalable.

### 🔹 1. Fork & Clone
- Fork the repository
- Clone it locally

### 🔹 2. Create a New Branch (MANDATORY)

Branch name format:

✅ Example:
- naveen/03-05-2026-two-sum
- john/15-04-2026-binary-search


---

### 🔹 3. Add Your Solution

- Create a new folder for your problem
- Add your solution file(s)
- Ensure clean and readable code

---

### 🔹 4. Commit Message Format

Your commit message **must clearly describe the change**:
Added: Two Sum solution using hashmap
Improved: Optimized binary search implementation
Fixed: Edge case for negative numbers

---

### 🔹 5. Push Your Branch

git push origin your-branch-name


---

### 🔹 6. Create Pull Request

- Open a PR to the `main` branch
- Provide:
  - Problem description
  - Approach used
  - Time & space complexity (if possible)

---

## ⚠️ Rules to Follow

- ❌ Do NOT push directly to `main`
- ✅ Always use a new branch
- ✅ Follow naming conventions strictly
- ✅ Keep code clean and readable
- ✅ Avoid duplicate problems (check before adding)
